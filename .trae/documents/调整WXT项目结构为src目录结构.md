# 调整WXT项目结构为src目录结构

## 目标
将当前的扁平结构（不带src/）调整为WXT官方推荐的src/目录结构，确保符合官方规范。

## 步骤

### 1. 创建目录结构
- 创建 `src/` 目录
- 在 `src/` 下创建以下子目录：
  - `assets/` - 静态资源文件
  - `components/` - 可复用组件
  - `hooks/` - React hooks
  - `entrypoints/` - 扩展入口点
  - `utils/` - 工具函数

### 2. 移动文件
- 将现有的 `entrypoints/` 目录移动到 `src/` 目录下
- 确保所有文件都正确放置在新的目录结构中

### 3. 修改配置文件
- 更新 `wxt.config.ts`，添加 `srcDir: 'src'` 配置
- 确保其他配置文件（如 `tsconfig.json`）正确指向新的目录结构

### 4. 调整导入路径
- 更新所有文件中的导入路径，确保它们正确指向新的目录结构
- 特别注意服务文件和组件文件之间的相对路径

### 5. 验证项目
- 运行 `npm run build` 确保项目能够成功构建
- 检查所有功能是否正常工作

## 预期结果
- 项目结构符合WXT官方推荐的规范
- 所有功能保持不变
- 代码组织更加清晰，便于维护

## 注意事项
- 确保所有导入路径都正确更新，避免路径错误导致构建失败
- 保持原有的功能实现不变，只调整目录结构
- 遵循WXT官方的命名规范和最佳实践
