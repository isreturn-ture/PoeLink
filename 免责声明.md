## 第一部分：产品行为与数据流说明

本部分依据本扩展实际代码与业务逻辑编写，用于准确说明扩展“做了什么、不做什么”，便于你评估合规与风险。

### 1.1 扩展实际实现的功能

| 功能 | 行为说明 | 数据去向 |
|------|----------|----------|
| 后端代理请求 | 将弹窗/内容脚本发起的 API 请求，由扩展后台代为转发至**你配置的后端服务地址**，以避免跨域限制；扩展不解析、不存储请求内容，仅做转发。 | 请求与响应在你配置的后端与扩展之间传输 |
| Cookie 同步 | 在你有意开启的前提下，将浏览器中**当前标签页及你配置的运管域名**下的部分 Cookie（如 JSESSIONID、accessToken、opsAccessToken）读取并同步至**你配置的后端**；后端可能据此代表你访问 RCS/运管等系统。 | 仅发往你配置的后端，扩展不向其他第三方发送 |
| 聊天 / AMR 排查辅助 | 你在扩展弹窗内输入的内容，经可选意图与实体识别后，连同识别结果一并发送至**你配置的后端的 `/api/chat` 等接口**；后端负责与业务系统交互并返回结果，扩展仅展示后端返回的文案、任务流水、日志下载链接等。 | 输入与识别结果发往你配置的后端；若你配置了 LLM，部分输入会发往你选择的 LLM 服务商 |
| 可选 LLM 调用 | 若你在配置中填写了 LLM API Key 与提供商（如 Moonshot/OpenAI/SiliconFlow），扩展会代你向该提供商的接口发送**你当次输入内容**，用于意图识别与实体抽取；扩展不存储你的 API Key 到自身服务器，仅本地使用。 | 你输入的文本会发往你选择的 LLM 服务商，请遵守该服务商的使用与隐私政策 |
| 健康检查与配置校验 | 对你配置的后端地址发起健康检查、配置校验、日志下载等请求，用于配置页“测试连接”等能力。 | 仅与你配置的后端通信 |
| 悬浮窗 / 弹窗 UI | 在浏览器任意页面注入悬浮球与弹窗界面，便于在不打开扩展独立弹窗时使用 PoeLink；不针对任何特定站点做 DOM 自动化或数据抓取。 | 无额外数据外发，仅本地 UI 与本地存储 |

### 1.2 扩展明确不实施的行为

- **不直接连接或操作** RCS、OPS、IWMS 等业务系统的前端页面或 DOM。  
- **不实施**对任何第三方网站或系统的界面自动化（如自动点击、自动填表、爬取页面结构）。  
- **不破解、不绕过**任何登录、授权或技术保护措施；与业务系统的交互（若有）由**你配置的后端**完成，扩展仅与该后端通信。  
- **不反编译、不反向工程**任何权利方软件；扩展代码为独立编写，不包含权利方源代码或目标代码。  
- **不向未披露的第三方**上传你的聊天内容或 Cookie；仅按上述数据流发往你配置的后端或你选择的 LLM 服务商。

### 1.3 你的配置与责任

- 你填写的**后端服务地址、数据库、运管地址、LLM API Key**等均保存在**本地浏览器存储**中，扩展不会主动上传至开发者服务器。  
- 你需自行确保：该后端由你合法控制或已获授权使用，其与 RCS/运管/第三方系统的对接符合相关许可与法律法规。  
- 使用本扩展即表示你已知晓上述数据流与责任边界。
